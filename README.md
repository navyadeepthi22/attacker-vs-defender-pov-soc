# Attacker vs Defender POV – SOC Case Study

## Overview
This project analyzes a simulated brute-force authentication attack from both the attacker’s and defender’s perspectives. The objective is to understand how attackers attempt credential compromise and how SOC analysts detect, analyze, and respond to such activity using logs and alerts.

This MITRE ATT&CK–driven case study helps bridge the gap between offensive techniques and defensive detection.

---

## Project Objective
To develop a deeper understanding of attacker behavior and SOC detection logic by analyzing the same attack scenario from two opposing viewpoints.

---

## Skills Demonstrated
- Adversary mindset analysis
- SOC alert triage and investigation
- Windows authentication log analysis
- Brute-force detection logic
- MITRE ATT&CK mapping
- Incident classification and escalation
- Defensive gap analysis

---

## Scenario Summary
A simulated brute-force login attempt is analyzed to understand:
- How attackers attempt credential access
- What assumptions attackers make about defenses
- How SOC analysts detect and respond to the activity

---

## MITRE ATT&CK Reference
- **Tactic:** Credential Access  
- **Technique:** Brute Force (T1110)

---

## SOC Outcome
**True Positive** – Activity escalated to Tier-2 SOC for further investigation and response.
